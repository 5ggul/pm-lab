# Polymarket 아비트라지 스캐너 (1단계)

멀티아웃컴(negRisk) 이벤트에서 아웃컴 확률 합이 100%에서 벗어난 틈을 찾아
Gamma API로 1차 스크리닝 → CLOB 실오더북으로 체결 가능 여부·실제 차익을 검증하는 스캐너.

## 아비트라지 등급

| 등급 | 타입 | 조건 | 성립 근거 |
|------|------|------|----------|
| 구조적 | SHORT_ALL | 전체 YES bid 합 > $1 → 모든 아웃컴 NO 매수 | negRisk("최대 하나만 YES") 구조만으로 확정 |
| 조건부 | LONG_ALL_YES | 전체 YES ask 합 < $1 → 모든 아웃컴 YES 매수 | 아웃컴 목록이 전체를 커버(exhaustive)해야 확정 — **수동 확인 필수** |
| 가짜(제외) | LONG_ALL_YES, edge > 0.05 | 합이 1에서 크게 모자람 | 사실상 비망라 이벤트(나열 안 된 결과로 끝날 확률이 큼) |

## 사용법

```
python scanner.py                                  # 1회 스캔
python scanner.py --loop 300                       # 5분 간격 무한 스캔
python scanner.py --min-liquidity 20000 --size 100 --verify-limit 8
```

주요 옵션: `--buffer` 가스/슬리피지 버퍼(기본 0.005), `--size` 검증 세트 수량(기본 100주),
`--top` near-miss 표시 수, `--max-events` 수집 상한.

## 출력

- 콘솔: 구조적/조건부 아비트라지 건수 + 오더북 실체결 검증 결과(실 edge, 필요자본, 기대차익)
- `opportunities.jsonl`: 스캔마다 기회 레코드 누적 (2단계 페이퍼 트레이딩 입력용)

## 2단계: 페이퍼 트레이딩 (실주문 없음)

```
python -u paper_trader.py --interval 300     # 5분 간격 상시 감시 + 가상 체결
python report.py                             # 장부 요약 (건수/자본/고정수익/자금잠김)
```

- `paper_ledger.jsonl` : 가상 체결 장부 (kind=trade) + 검증 탈락 기록 (kind=rejected)
- `scan_stats.jsonl`   : 스캔마다 시장 전반 기회 밀도 기록
- `paper_state.json`   : 중복 booking 방지 상태 (같은 이벤트+타입은 1회만 기록)

아비트라지는 진입 시점에 손익이 확정되므로, 오더북 시뮬레이션이 성공한 시점의
실체결 edge로 수익을 장부에 고정한다. 목적은 "하루에 몇 건 / 총 얼마"의 측정.

## 대시보드

```
python dashboard.py          # http://localhost:8899 (30초 자동 갱신)
```

paper_ledger.jsonl / scan_stats.jsonl을 읽어 KPI(가상 체결 수·고정 수익·필요 자본·ROI·자금 잠김),
누적 수익 차트, 스캔별 기회 밀도 차트, 체결 내역 테이블을 보여준다. 표준 라이브러리만 사용
(차트는 CDN의 Chart.js). 페이퍼 트레이더와 동시에 띄워두면 실시간 모니터링이 된다.

## 3단계: 단기 Up/Down 마켓 워처 (페이퍼 전용)

바이럴 글들이 주장하는 5분/15분 크립토 Up/Down 봇 엣지가 우리 폴링 속도에서
실존하는지 측정. 실주문 없음.

```
python -u updown_watcher.py --assets btc,eth --windows 300,900   # 상시 측정
python updown_report.py                                          # 결과 요약
```

측정 3축:
1. **시세지연 괴리**: Binance 실시간가 기반 공정확률(정규분포 근사) vs 폴리마켓 VWAP
   실체결가. edge 임계 초과 시 가상 진입 → 윈도우 종료 시 Binance 종가로 정산해
   승률·손익 측정. (실정산은 Chainlink 기준 — 근사 오차 캐비앳)
2. **시점분리 페어(temporal arb)**: 윈도우 내 Up 최저가 + Down 최저가 합 최소값 기록.
   $1 미만 빈도 = "시간을 나눠 양쪽을 샀다면" 가능했던 세트 빈도 (방향 리스크의 대가).
3. **마감 스나이핑 관측**: 마감 15초 전 우세 아웃컴이 99¢ 미만인 빈도 기록만 (진입 안 함).

산출: `updown_trades.jsonl`(가상 체결·정산), `updown_windows.jsonl`(윈도우 통계),
`updown_positions.json`(미정산 상태).

주의: `--fee` 기본 0.01/주는 가정값. 단기 크립토 마켓의 실제 테이커 수수료는 실거래 전 확인 필수.

## 2026-07-13 첫 스캔 결과 (현실 체크)

- 유동성 $2만+ 이벤트 600개 중 구조적 아비트라지 13건 검출 — **존재는 하지만 경제성이 없음**
  - 예: NBA 2027 챔피언 SHORT_ALL → 자본 $2,898 묶어서 기대차익 $1.60 (약 0.06%, 2027년 정산까지 잠김)
  - 예: Fed 9월 결정 → 자본 $399에 $0.70 (약 0.18%, 2개월)
- 조건부(LONG_ALL_YES)는 1.8~3.5% edge가 보이나 망라성 수동 확인 필요
  (Fed 금리 구간·주지사 수 구간처럼 버킷형 마켓은 망라적이라 실제 기회일 수 있음)
- 결론: 단순 스캔형 아비트라지는 봇 경쟁으로 이미 소진. 남은 틈은 자본 대비 수익이 미미.
  엣지를 찾으려면 속도(신규 마켓 개설 직후), 조건부 기회의 자동 망라성 판정,
  또는 크립토 단기 마켓 vs 거래소 시세 괴리 같은 다른 축이 필요.
